# Sumo-Guard Sequence Diagram (Data Analysis Flow)

This diagram illustrates the sequence of events when a user views their health predictions. It shows how the frontend interacts with Firestore and the internal prediction engine.

## Flow Description

1.  **Trigger**: The sequence starts when the user enters the Predictions page or when a new log is added.
2.  **Data Retrieval**: The application concurrently fetches three streams of data from Firestore:
    *   **Sleep Logs**: Time-series data of sleep quality and duration.
    *   **Habit Logs**: Daily lifestyle tracking (exercise, water, stress).
    *   **Health Profile**: Static physiological data (age, weight, height) and family history.
3.  **Processing**: The `analyzePredictions` function (local to the client) acts as the logic core. It processes the raw data into formatted risks and scores.
4.  **Presentation**: The UI conditionally renders based on whether the `dataInsufficient` flag is returned, ensuring users aren't shown inaccurate predictions based on too little data.
