## Detailed Flow Explanation

### 1. Authentication
The application uses **Firebase Auth** to ensure that health data is private and scoped to the specific user. Every subsequent database request includes the user's UID.

### 2. Data Persistence (Habit Logging)
When a user logs habits:
*   The frontend uses `setDoc` with `merge: true` to either create or update the log for that specific date.
*   Data points include exercise minutes, water intake, stress levels, smoking, alcohol, and meditation.

### 3. Prediction Orchestration
The **Predictions Engine** is a client-side logic layer. When the user views predictions:
*   The frontend fetches historical data (up to the last 50 entries) for both habits and sleep.
*   It fetches the user's static profile (age, weight, height, family history).
*   The `analyzePredictions` function runs locally, ensuring low latency and privacy (data doesn't leave the client for processing).

### 4. Risk Calculation Logic
The engine evaluates:
*   **Sleep Quality**: Duration vs. Bedtime consistency.
*   **Habit Impact**: Correlates exercise/stress with known risk factors.
*   **Physiological Risks**: BMI calculation and family history weighting.
