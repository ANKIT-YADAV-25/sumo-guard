## Component Breakdown

### 1. Frontend System (React/Vite)
- **Vite**: Serves as the build tool and development server for high-performance frontend delivery.
- **React UI**: Built with a component-based architecture using modern hooks and functional components.
- **Tailwind CSS**: Provides the styling layer for a responsive, modern interface.
- **React Query**: Handles the caching and synchronization of server-side data (Firestore) to minimize redundant network requests.

### 2. Analytical Core (Prediction Engine)
- **Client-Side Logic**: To ensure privacy and speed, the risk analysis algorithms run directly on the user's device. 
- **Data Modelling**: Transforms raw logs into structured disease risk profiles using physiological metrics and clinical heuristics.

### 3. Backend-as-a-Service (Firebase)
- **Firebase Auth**: Manages secure user registration, login, and session persistence.
- **Cloud Firestore**: A real-time NoSQL database that stores user profile details and time-series logs (Sleep/Habits) in a hierarchical document structure.
- **Security Rules**: Ensure that users can only access their own data sub-collections.

### 4. Native Integration (Capacitor)
- **Capacitor**: Wraps the web application into a native Android container, allowing it to be deployed as an APK and potentially interact with native device sensors in future iterations.
