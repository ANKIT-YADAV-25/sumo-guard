## Use Case Descriptions

### 1. User Authentication (UC1)
*   **Actor**: User
*   **Description**: The user signs up or signs in using Firebase Auth to access their private health data.

### 2. Daily Health Logging (UC3, UC4)
*   **Actor**: User
*   **Description**: The user inputs data regarding their last night's sleep (bedtime, wake time, quality) and daily habits (exercise, water, stress, smoking, alcohol). 
*   **Post-condition**: Data is stored in Firestore keyed by the current date.

### 3. Health Analysis & Predictions (UC6)
*   **Actor**: User, Prediction Engine
*   **Description**: The system aggregates historical logs and profile data to calculate risk levels for various chronic diseases (Cardiovascular, Diabetes, Mental Health, etc.).
*   **Requirement**: Requires at least 3 days of logs for improved accuracy.

### 4. Personalized Recommendations (UC7)
*   **Actor**: User, Prediction Engine
*   **Description**: Based on high-risk factors identified in the analysis, the system provides actionable medical and lifestyle advice (e.g., "Reduce sodium intake", "Increase daily walking").

### 5. Profile & Onboarding (UC2, UC8)
*   **Actor**: User
*   **Description**: The user maintains their physical metrics (age, height, weight) and medical history, which serve as the baseline for all health predictions.
